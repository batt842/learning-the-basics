# CAP theorem
CAP theorem is a concept that a distributed system can only have 2 of 3: Consistency, Availability and Partition Tolerance.

CA: RDBMS
CP: HBase, MongoDB, Redis, ...
AP: CouchDB, Cassandra, , Dynamo, ...
